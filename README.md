## &nbsp; About Me
I'm an engineer who likes to understand **how things actually work** — not just how to use them.
Most of my work sits at the intersection of **backend engineering** and **artificial intelligence**. I build systems where frontends, APIs, databases, ML models and LLMs work together as a single coherent product — not as isolated experiments.
I enjoy breaking complex systems into small, understandable components. I'm more interested in how distributed services, ML inference pipelines and AI agents communicate than in simply making an application run.
Currently, I'm spending more time learning how **production AI systems** are designed — model serving, retrieval-augmented generation, agentic workflows — rather than focusing only on training loops.
```
→  I don't collect frameworks. I solve problems with whatever tools make sense.
→  I think about latency, reliability and system design before writing the first line.
→  I prefer building complete systems over polishing isolated features.
```
